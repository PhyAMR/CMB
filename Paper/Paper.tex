\begin{abstract}
In this article, we explore the mathematical and physical framework for
analyzing the Cosmic Microwave Background (CMB) angular correlation
function. We demonstrate how this function and its derived statistics
can be used to test the standard \(\Lambda\)CDM cosmological model,
potentially revealing new tensions, particularly concerning the
assumption of a flat geometry.
\end{abstract}





\end{frontmatter}
    

\section{Introduction}\label{introduction}

The goal of this text is to explain what I have done in order to make
sure the statistical comparisons presented below are reproducible,
robust and clearly traceable to the data and code used. This manuscript
documents the computational pipeline, the choices made for data
processing, and the statistical tests performed to assess the
consistency between observed Planck multipoles and theoretical
expectations.

The work aims to (1) describe the correlation-function based statistics
used to probe large-angle anomalies in the cosmic microwave background
(CMB), (2) present a reproducible implementation using CAMB, Monte Carlo
realizations and Planck-style simulations, and (3) provide clear
instructions so readers can reproduce and extend the results using the
code in the repository. The emphasis is on transparency: every derived
number can be traced to an input file or to a well-documented routine in
the functions/ directory.

\subsection{Physics background and
intuition}\label{physics-background-and-intuition}

This subsection provides the physical context and intuition behind the
statistics used in the analysis so a reader unfamiliar with the
implementation can understand why the chosen diagnostics are
informative.

The cosmic microwave background (CMB) temperature field on the sphere is
a realization of a random process that is to very good approximation
statistically isotropic and (to leading order) Gaussian. Its statistical
information is compactly encoded in the angular power spectrum
\(C_{\ell}\), defined by the ensemble average of spherical-harmonic
coefficients \(a_{\ell m}\):
\(\langle a_{\ell m}a^{*}_{\ell' m'}\rangle = C_{\ell}\,\delta_{\ell\ell'}\,\delta_{mm'}\).
In practice we work with the commonly used scaled quantity
\(D_{\ell}=\ell(\ell+1)C_{\ell}/(2\pi)\) which emphasizes the
contribution per logarithmic interval in multipole.

Large-angle anomalies are features visible at low multipoles
(\(\ell\lesssim 30\)) where cosmic variance (the intrinsic sample
variance because we observe only one sky) is large. A direct comparison
of observed multipoles to theory must therefore account for the full
sampling distribution of the estimated multipoles and derived
quantities. This motivates the use of Monte Carlo and pipeline
simulations that reproduce the estimator behaviour and the effects of
masks, beams and noise.

The two-point correlation function \(C(\theta)\) is the harmonic-space
power spectrum transformed back to real (angular) space by a sum over
Legendre polynomials. Because it is a sum over a wide range of \(\ell\),
the correlation function is often more intuitive for discussing angular
separations: features at large angles receive contributions from many
low-\(\ell\) modes and are therefore sensitive to coherent deviations at
the largest scales.

Two complementary statistics are emphasised in this work:

\begin{itemize}
\tightlist
\item
  The interval-averaged correlation \(\langle\xi\rangle_a^b\) is a
  linear functional of \(C(\theta)\) that summarizes the mean
  correlation within a chosen angular window. Being linear in the data,
  it is directly sensitive to shifts in the mean correlation and is
  relatively straightforward to propagate errors for.
\item
  The quadratic statistic \(S_a^b\) (including the classic \(S_{1/2}\)
  choice) integrates the square of the correlation over an interval.
  Because it is quadratic in the map (or in \(C(\theta)\)), it is
  sensitive to the variance of correlations and can emphasize deviations
  that are small in mean but large in dispersion across realizations.
  The matrix \(T_{nm}\) that appears in the analytic expression encodes
  the angular overlap of Legendre modes within the interval and must be
  computed accurately for robust error estimates.
\end{itemize}

An additional simple but useful diagnostic is \(C_{180}\), the antipodal
correlation, which probes correlations between widely separated points
and is comparatively insensitive to small-scale masking effects.

Why two types of statistics? The linear \(\langle\xi\rangle\) is less
affected by distributional skew or amplified tails and often provides a
stable test for mean shifts, while the quadratic \(S\) statistic can
highlight broader classes of departures from Gaussian predictions (for
example excess variance or coherent real-space structure). Using both
statistics in parallel gives a more complete picture of possible
anomalies.

Role of simulations and modelling: to assess whether an observed
statistic is anomalous one must know its sampling distribution under the
null hypothesis (usually the standard cosmological model). There are
three practical ways to generate such distributions in the code:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Per-\(\ell\) Gaussian sampling (fast): draw independent Gaussian
  perturbations of each reported \(D_{\ell}\) using reported
  per-\(\ell\) errors and compute derived statistics. This approach is
  useful for quick checks but neglects estimator covariance introduced
  by masking and beam corrections.
\item
  Planck-style forward pipeline (realistic): generate full-sky
  realizations from a theoretical \(C_{\ell}\), apply beam and pixel
  windows, add noise, and run the same estimator as used on the real
  data. This reproduces measurement biases and mode coupling induced by
  the mask and is therefore the preferred route when available.
\item
  MCMC ensembles (theory sampling): when posterior samples of
  cosmological parameters are available (Planck chains), converting each
  posterior sample to a CAMB spectrum and then to \(D_{\ell}\) produces
  an ensemble that captures theoretical uncertainty propagated from
  parameter estimation.
\end{enumerate}

Practical interpretation and caveats: p-values reported in this work are
empirical: for each statistic we compare the observed value to the
distribution derived from the chosen ensemble and report the fraction of
simulations as extreme (two-sided) as the observation. Important caveats
include the ``look-elsewhere'' effect (multiple, correlated statistics
increase the chance of finding at least one low p-value by chance) and
the strong influence of cosmic variance at low \(\ell\). Therefore an
anomalously low p-value should be interpreted as a prompt for further
investigation rather than definitive evidence for new physics.

This subsection aims to equip a reader with the conceptual understanding
needed to follow the code mapping: the next section (Methodology)
documents how these statistics are implemented and where to find the
corresponding routines in the repository.

\section{Methodology}\label{methodology}

This section describes the data sources, the numerical implementation,
and the exact sequence of processing steps required to reproduce the
reported results. The repository contains a set of scripts and a
directory structure (for example \texttt{run\_\{lmax\}\_\{n\}}) that
collect simulation outputs, theory chains and auxiliary files; the
Methods below explain which files are produced and how they are used by
the plotting and analysis routines.

We emphasize two practical points that improve reproducibility for users
of this repository:

\begin{itemize}
\tightlist
\item
  All Monte Carlo and chain outputs are saved as human-readable or
  standard binary arrays (e.g.~.npy, .txt) in the run directory; when
  present the plotting functions will read these files rather than any
  intermediary pickle files.
\item
  A boolean flag (use\_recomputed\_statistics) is provided to instruct
  the plotting utilities to use statistics recomputed from a replacement
  D\_ell file (the recomputed files live under
  run\_dir/recomputed\_statistics). By default this flag is false,
  matching the principle that analysis should reproduce published
  results unless a replacement dataset is intentionally provided.
\end{itemize}

The remainder of this subsection provides details about the specific
statistics and the files produced by each stage.

\subsection{Functions and statistics}\label{functions-and-statistics}

The analysis relies on three related observables computed from a power
spectrum \(D_{\ell}\) (equivalently \(C_{\ell}\)):

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  The two-point correlation function \(C(\theta)\), which probes
  temperature correlations as a function of angular separation.
\item
  The interval-averaged correlation statistic \(\langle\xi\rangle_a^b\),
  a linear functional of \(C(\theta)\) that summarizes the mean
  correlation within an angular interval \([\theta_a,\theta_b]\)
  (expressed below in terms of \(a=\cos\theta_b\), \(b=\cos\theta_a\)
  for convenience).
\item
  The integrated squared-correlation \(S_a^b\) (originally \(S_{1/2}\)
  when \(a=\cos(60^\circ)\) and \(b=\cos(180^\circ)\)), which emphasizes
  power on a given angular range and is quadratic in the spectrum.
\end{enumerate}

Below we summarise each quantity and state how it is computed and stored
in the repository outputs.

\textbf{Correlation function.} The correlation function is defined as

\[C(\theta)=\sum_{\ell}\frac{2\ell+1}{2\ell(\ell+1)}D_{\ell}P_{\ell}(\cos\theta)\]

where \(D_{\ell}=\ell(\ell+1)C_{\ell}/(2\pi)\) in the conventions used
here. In practice the summation runs from \(\ell=2\) to a configured
\(\ell_{\max}\) (the code parameter \texttt{lmax}). The implementation
used to compute \(C(\theta)\) is provided in Appendix A, Listing 1; the
function supports both direct use of a CAMB-produced theoretical
spectrum and Monte Carlo realizations produced by the Planck-style
pipeline.

Computed outputs and storage:

\begin{itemize}
\tightlist
\item
  Per-realization correlation arrays are saved as \texttt{Corr.npy} in
  the simulation or model directory.
\item
  The mean correlation used for plotting and comparison is written as
  the mean across realizations and also exposed via the plotting helpers
  (see functions/plots.py).
\end{itemize}

\textbf{Interval-averaged correlation \(\langle\xi\rangle_a^b\).} The
interval average is defined by

\[\langle\xi\rangle_a^b = \frac{\int_a^b C(\theta) \, d\cos\theta}{b-a} = \sum_{\ell} \frac{D_{\ell}}{2\ell(\ell+1)}\frac{1}{b-a}\left[P_{\ell+1}(\cos\theta)-P_{\ell-1}(\cos\theta)\right]_{a}^{b}\]

This is implemented in the repository (see Appendix A, Listing 2) and
the per-realization values are saved as columns in
\texttt{xiv\_statistic.txt} (with a corresponding \texttt{columns.txt}
describing the order).

\textbf{\(C_{180}\) and \(S_a^b\).} The \(C_{180}\) statistic is simply
\(C(180^{\circ})\), i.e.~the antipodal correlation; it is robust to
certain mask effects because it samples the largest angular separation.
The quadratic statistic is defined as

\[S_a^b = \int_a^b C(\theta)^2\sin\theta\, d\theta = \sum_{n,m} A_{n}A_{m} D_n D_m\, T_{nm}\]

where \(T_{nm}\) are overlap integrals of Legendre polynomials over the
interval. The code computes the \(T_{nm}\) matrix analytically (see
Appendix A, Listing 3) and writes per-realization \(S\) values to
\texttt{S\_statistics.txt} and a separate file
\texttt{C180\_statistics.txt} for the antipodal values.

Implementation notes:

\begin{itemize}
\tightlist
\item
  The code carefully handles edge cases where single-realization outputs
  reduce to 1D arrays and ensures consistent column ordering via
  \texttt{columns.txt} files.
\item
  All text outputs accompanying a run are plain ASCII tables
  (\texttt{.txt}) so they can be reloaded by other tools without
  requiring the Python source.
\end{itemize}

\subsection{Experimental values}\label{experimental-values}

The experimental inputs used throughout the analysis are taken from the
Planck legacy archive and the associated likelihood releases compiled by
the Planck Collaboration. Concretely, the quantities used are the binned
and deconvolved temperature multipoles \(D_{\ell}\) (or equivalently
\(C_{\ell}\)) and their reported uncertainties. When reproducing these
results the following practical choices were applied:

\begin{itemize}
\tightlist
\item
  \textbf{Source and versioning:} the specific Planck chains and central
  best-fit products are documented in the repository's
  \texttt{README.md} and were downloaded from the Planck Legacy Archive.
  Users should verify the exact file timestamps if precise bitwise
  reproducibility is required.
\item
  \textbf{Multipole selection:} for large-angle statistics the low
  multipoles (\(\ell\lesssim 30\)) carry most of the signal; however the
  pipeline accepts any configured \(\ell_{\max}\) and the scripts record
  the value used in the run directory.
\item
  \textbf{Uncertainties and mask treatment:} the Planck collaboration
  provides covariance or per-\(\ell\) error estimates depending on the
  data product. In this work we treat reported multipole errors as
  Gaussian 1\(\sigma\) uncertainties when generating naive per-\(\ell\)
  Monte Carlo samples (see Appendix A). For more realistic simulations
  the Planck-style pipeline (\texttt{functions/simulation.py})
  propagates mask, beam and noise effects explicitly when
  \texttt{run\_pipeline=True}.
\item
  \textbf{Reproducible ingest:} the function
  \texttt{Data\_loader.experimental\_values(...)} in the codebase
  centralizes the parsing of experimental inputs and returns a
  dictionary of named statistics (e.g.~\texttt{s12\_60\_180} or
  \texttt{xiv\_...}) that the plotting routines use. This ensures
  consistent ordering and naming between experimental and simulated
  columns.
\end{itemize}

\subsection{Theory and simulations}\label{theory-and-simulations}

\subsubsection{Theory}\label{theory}

Theoretical predictions are generated with CAMB. The code calls CAMB
with a vector of cosmological parameters sampled either from a best-fit
vector or from an MCMC chain, then extracts the temperature power
spectrum and converts it to the \(D_{\ell}\) convention used throughout
the repository. The CAMB calls are implemented in
\texttt{functions/cosmo.py} (see Listing 1).

Parameters: the full set of sampled parameters is listed below and
matches the labels used in the Planck chains. When a chain is available
the code reads the last \(N\) samples (by default 1000) to form an
ensemble of theoretical spectra; when only a best-fit entry is used the
code instead samples per-\(\ell\) Monte Carlo realizations around the
reported \(D_{\ell}\) (see below).

\begin{itemize}
\tightlist
\item
  \texttt{ombh2}, \texttt{omch2}, \texttt{H0}, \texttt{omk},
  \texttt{YHe}, \texttt{nnu}, \texttt{nrun}, \texttt{Alens},
  \texttt{ns}, \texttt{As}, \texttt{w}, \texttt{wa}, \texttt{mnu},
  \texttt{tau}
\end{itemize}

\subsubsection{Simulation pipelines}\label{simulation-pipelines}

Two complementary strategies are provided to obtain ensembles of
\(D_{\ell}\) and derived correlation statistics:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  \textbf{Per-\(\ell\) Monte Carlo (fast, approximate).} Starting from a
  vector of reported multipoles and their 1\(\sigma\) errors, the code
  draws independent Gaussian realizations for each \(\ell\) (or uses the
  per-\(\ell\) sampling that simulates the underlying \(a_{\ell m}\)
  modes when available). This method is computationally cheap and useful
  to estimate sampling distributions under the simplifying assumption of
  independent multipoles; its implementation appears in Listing 2 and
  the results are written to \texttt{simulation/D\_ell.npy},
  \texttt{simulation/Corr.npy}, and the scalar statistic files
  (\texttt{S\_statistics.txt}, \texttt{xiv\_statistic.txt},
  \texttt{C180\_statistics.txt}) together with \texttt{columns.txt}.
\item
  \textbf{Planck-style pipeline (detailed, realistic).} The code can
  instead run a Planck-like simulation that generates full-sky
  realizations of the CMB given a theoretical \(C_{\ell}\), applies the
  instrumental beam and pixel window, adds a noise model, and estimates
  \(D_{\ell}\) from the simulated sky maps using the same estimator as
  in the Planck pipeline. This yields realistic covariance between
  multipoles induced by masking, beam convolution and noise. The
  parallel implementation is available in
  \texttt{functions/simulation.py} (Listing 3) and stores outputs in the
  same directory structure as above for downstream analysis.
\end{enumerate}

\textbf{MCMC-based ensembles.} When a parameter chain (getdist or plain
text chains) is available the script \texttt{chain\_results(...)} (in
\texttt{functions/cosmo.py}) fetches the requested final segments of the
chain, converts each sample to CAMB cosmological parameters, computes
the theoretical spectrum and correlation for each sample, and stores the
per-sample arrays under
\texttt{theory/\textless{}model\textgreater{}/D\_ell.npy} and
\texttt{theory/\textless{}model\textgreater{}/Corr.npy}. The
corresponding scalar statistics for each sample are written to
\texttt{theory/\textless{}model\textgreater{}/S\_statistics.txt},
\texttt{xiv\_statistic.txt} and \texttt{C180\_statistics.txt} and a
\texttt{columns.txt} file is included to guarantee consistent ordering.

\textbf{Run directory layout and bookkeeping.} Each analysis run is
organized under a directory named \texttt{run\_\{lmax\}\_\{n\}} where
the simulation outputs and theory models are stored in the subfolders
\texttt{simulation/} and
\texttt{theory/\textless{}model\textgreater{}/}. When the optional
recompute step (see \texttt{recompute\_statistics} in \texttt{main.py})
is invoked it writes replacement statistic files to
\texttt{run\_.../recomputed\_statistics/}; the plotting utilities expose
a flag \texttt{use\_recomputed\_statistics} that directs them to read
these recomputed files if desired. This separation keeps original run
outputs intact while allowing straightforward replacement for dedicated
tests.

\textbf{Outputs and reproducibility notes.}

\begin{itemize}
\tightlist
\item
  All numeric arrays are saved as \texttt{.npy} where possible (fast
  binary reload) and scalar tables as ASCII \texttt{.txt} to ease
  inspection.
\item
  The repository scripts record key metadata (\texttt{lmax}, number of
  realizations, random seeds when used) inside the run folder so
  reported figures can be traced back to the exact inputs.
\end{itemize}

Following the generation of theoretical and simulated ensembles, the
pipeline computes the statistics described above for every realization
and uses those distributions to assign empirical p-values when comparing
to the experimental measurements.
